# Authoring 3D Models + Textures for Concord

This doc covers how to feed authored 3D content into Concordia such that it
seeds the **procedural + evo systems** — meaning every model and texture you
drop in becomes the seed for thousands of derivative variants generated by the
world's emergent simulation, and you collect perpetual royalties through the
cascade.

## The leverage shape

```
1 authored oak-tree model (.glb)
  ↓ inserted as evo_asset(kind='mesh', source='authored', tags=['seed'])
  ↓ L-system tree generator uses it as the leaf-cluster geometry
  ↓ spawns 1000 variant trees biased by biome / season / lighting
  ↓ each variant gets minted as evo_asset(source='evolved', lineage→oak_tree)
  ↓ NPCs interact (gather wood, shelter, ambush)
  ↓ certain variants get more interactions = "selected"
  ↓ top variants promoted to canonical pool
  ↓ royalty cascade pays you for every derivative citation, 50 gens deep
```

You author once. The world generates the variants. The economy pays you forever.

## Quickstart — get content into the world in 30 minutes

### 1. Bulk-fetch CC0 packs (one-time)

```bash
# Default — Poly Haven + AmbientCG + OS3A (~5 minutes, ~500MB)
node scripts/fetch-cc0-assets.mjs

# All — including Quaternius packs (~30 minutes, ~3GB)
node scripts/fetch-cc0-assets.mjs --all
```

Files land in `content/world/_shared/{models,textures,hdris}/`. The fetcher
verifies CC0 license per source and refuses to download anything else.

### 2. Hand-drop your own work

```bash
cp ~/Blender/output/my_dragon.glb         content/world/_shared/models/authored/
cp ~/Krita/exports/hand_painted_wood.png  content/world/_shared/textures/authored/
```

Naming convention: `kind_name_descriptor.ext`
- `tree_oak_mature.glb`, `tree_pine_young.glb`
- `stone_cobble_albedo.png`, `stone_cobble_normal.png`, `stone_cobble_roughness.png`
- `building_townhouse_v01.glb`

### 3. Register them in evo_assets

```bash
# On a running server: just restart — the boot pipeline auto-scans
# the directory and registers everything at boot+30s.

# Without restarting:
node scripts/seed-evo-assets.mjs
```

### 4. Verify

```bash
# Browse the registry via lens macro
curl -X POST http://localhost:5050/api/lens/run \
  -H 'Content-Type: application/json' \
  -H "Cookie: concord_auth=$TOKEN" \
  -d '{"domain":"evo","name":"asset-stats"}'
```

All sources below are **CC0 / public domain** — no attribution required,
commercial use OK. The fetcher refuses to pull anything else.

|---|---|---:|---|

### Kenney + Quaternius bulk-install

These ship as ZIPs you extract yourself. Workflow:

```bash
# 1. Download from kenney.nl or quaternius.com
#    (Kenney mega-bundle ~2GB, Quaternius packs ~500MB each)
mkdir -p ~/concord-assets/{kenney,quaternius}
cd ~/concord-assets/kenney
unzip ~/Downloads/All-Kenney-NL-Assets.zip
cd ~/concord-assets/quaternius
for z in ~/Downloads/quaternius-*.zip; do unzip "$z"; done

# 2. Tell the server where they live
export KENNEY_BUNDLE_DIR=~/concord-assets/kenney
export QUATERNIUS_DIR=~/concord-assets/quaternius

# 3. Restart the server. Bootstrap will pick them up at T+30s.
```

## Hand-painted textures — the recommended toolchain

| Tool | Cost | Best for |
|---|---|---|
| **Procreate** (iPad) | $13 one-time | Painting albedo/diffuse on UV layouts. Easiest learning curve. |
| **Krita** (desktop) | Free | Full Photoshop-replacement for desktop painting. |
| **Substance Painter** | $20/mo Steam | Industry-standard 3D texture painting — paint directly on the model. |
| **Quixel Mixer** | Free (Epic) | Material mixing + smart materials. Pairs well with Megascans. |
| **Armor Paint** | Free (open source) | Substance Painter clone, runs anywhere. |

### Texture pipeline (hand-painted)

1. Model in Blender (free) → unwrap UVs → export `.glb` + UV layout PNG
2. Open UV layout in Procreate/Krita → paint albedo + normal + roughness
3. Export 3 textures (1024² or 2048²) → drop into `content/world/_shared/textures/authored/`
4. Naming: `{model_id}_albedo.png`, `{model_id}_normal.png`, `{model_id}_roughness.png`
5. Restart server → bootstrap registers + procedural pool picks up

### Texture pipeline (AI-assisted)

If you want to scale:
- **SDXL + ControlNet** (local, free if you have a GPU): text-to-PBR-texture
- **Substance 3D Sampler** ($20/mo): photo-to-PBR-texture
- **Polycam** ($8/mo): phone-LiDAR scan → mesh + texture
- **Tripo3D / Meshy.ai** ($20/mo): text-to-3D + texture in one shot

## Procedural seed-pool — how authored assets feed the generators

Once registered, the procedural systems pull from `evo_assets` automatically:

| Generator | Reads | Produces |
|---|---|---|
| `lib/world-lens/l-system-tree.ts` | Mesh assets tagged `tree`, `foliage` | 1000s of unique trees per biome |
| `lib/world-lens/procedural-buildings.ts` | Mesh assets tagged `building_module`, `wall`, `roof`, `door` | Cities + villages procgen |
| `lib/world-lens/rock-gen.ts` | Mesh assets tagged `rock`, `stone` | Terrain decorations |
| `lib/world-lens/biome-blend.ts` | Texture assets tagged `ground`, `terrain` | Per-biome ground shaders |
| `lib/world-lens/material-seed.ts` | Material assets (any source) | Material variant pool |
| `lib/world-lens/texture-forge.ts` | Texture assets | Procedural texture variations |

The generators read by **tag + category**, not by filename. Tag your assets
to control what the procgen pulls. Examples:

- `tree_oak_mature.glb` → tags `['authored', 'seed', 'tree', 'foliage', 'temperate']`
- `building_thatch_cottage.glb` → tags `['authored', 'seed', 'building_module', 'wall', 'medieval']`

(The `bootstrapAuthoredLocal` loader auto-tags with `['authored', 'seed']`.
For finer-grained tags, edit the row in `evo_assets` after registration or
extend the loader.)

## Evo cycle — how variants emerge from seeds

`server/lib/evo-asset/scheduler.js` runs the promotion pipeline:

1. **Quality gate** evaluates each authored seed (mesh-validity, UV-validity,
   normal-validity, scale-sanity).
2. **Refinement passes** generate variants from each seed (param shifts in
   the L-system tree generator, biome-tinted ground textures, etc.).
3. **Interaction signal** accumulates per-variant — NPCs harvesting, players
   citing, weather damaging, time-passing.
4. **Promotion** happens when a variant exceeds the seed's quality_level +
   interaction threshold. Promoted variants become the new canonical seed
   for their lineage.
5. **Royalty cascade** (`server/economy/royalty-cascade.js`) pays the
   original author every time a derivative is cited or transacted, halving
   per generation, capped at 50 generations + 0.0005 CC floor.

The author gets paid forever. The world generates infinite variety.
You authored once.

## Don't commit binaries to git

`content/world/_shared/.gitignore` excludes everything binary — git holds
only the directory structure + READMEs + LICENSE notes. Distribute large
asset packs separately:

- S3 bucket per environment
- Direct from CC0 source via `scripts/fetch-cc0-assets.mjs`
- BitTorrent for community-distributed seed packs
- IPFS for federation-friendly addressing

## Quick commands cheatsheet

```bash
# Bulk-fetch all CC0 sources (~3GB total)
node scripts/fetch-cc0-assets.mjs --all

# Only one source
node scripts/fetch-cc0-assets.mjs --source=polyhaven --limit=100

# Register everything that's already on disk (no server boot needed)
node scripts/seed-evo-assets.mjs

# Point at an arbitrary asset directory
node scripts/seed-evo-assets.mjs --dir=/path/to/my/blender/exports

# Point at extracted Kenney + Quaternius dirs
node scripts/seed-evo-assets.mjs \
  --kenney=~/concord-assets/kenney \
  --quaternius=~/concord-assets/quaternius

# Verify registry totals
sqlite3 server/data/concord.db \
  "SELECT source, kind, COUNT(*) FROM evo_assets GROUP BY source, kind"
```
