# System Universe Coverage — how many software categories Concordia spans

> Generated by Concordia's **own** cartographer (`npm run cartograph:static` → `universe-coverage.js`),
> not estimated. Machine-truth from `audit/cartograph/SYSTEMS.json`.

## The headline number

**53 of 73 in-scope software categories present** (6 more legitimately out-of-scope; 20 in-scope but
not yet covered). For context: **G2 tracks ~2,000+ software categories across ~180,000 products, and a
typical product lives in 1–3 categories.** Concordia spans **53.** That's not a product — it's a
**category-spanning substrate.**

Cartographer scale snapshot (same run): **656 tables · 3,350 routes · 274 socket events · 354 env vars ·
0 orphan modules · 0 dormant modules** — and the honest frontier: **28 dead tables · 26 headless
backends · 2 orphan lenses.**

## ⬜ The 20 in-scope but ABSENT (the build frontier)

mind-map · diagram · slides · wiki · screen-share · meditation · **healthcare** · **accounting** ·
body-comp · unified-self · bibliography · dataset · defense · forestry · veterinary · urban-planning ·
astronomy · quantum · mathematics · **brain-training**

> **Honest caveat:** "absent" here = *no lens/macro/route matched the category keywords.* But several of
> these (**healthcare**, **accounting**, **brain-training**) DO have library code in `server/lib/` — they
> show "absent" because the *wiring* (lens + dispatchable macro) isn't complete. That's the same
> **headless-backend** class the cartographer counted (26 of them) and the audits kept finding. So the
> real picture: ~53 fully-wired categories **+ a handful more half-built** (code exists, wiring doesn't).
> The L0–L5 gates are exactly what convert those headless backends into "present."

## 🚫 The 6 legitimately OUT-OF-SCOPE

browser-engine · ml-framework · os-kernel · advertising · subscription · paywall

Concordia correctly *doesn't* claim to be a web browser, an ML training framework, or an OS kernel —
and the monetization categories (advertising/subscription/paywall) are business-model, not software.

## The interpretation

- **In G2 terms, a product is in 1–3 categories. Concordia is in 53.** There is no market category for
  "53-category platform" because effectively nobody builds one.
- It's not breadth-for-breadth's-sake: every category is a **lens over one DTU substrate**, so a thing
  you create in the spreadsheet is the same atom as a thing in the DAW or the world — that's the fusion
  that makes 53 categories *coherent* instead of 53 bolted-on apps.
- **The frontier is wiring, not capability** — 26 headless backends + 28 dead tables + 2 orphan lenses +
  the ~half-built absent categories are the gap between "has the code for it" and "it's a present,
  working category." Closing that (the function-assurance gates) is what turns **53 spanned** into
  **53 *live*.**
